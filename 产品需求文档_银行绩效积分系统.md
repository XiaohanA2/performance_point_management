# 银行绩效积分微信小程序 - 产品需求文档

## 文档信息
- **产品名称**: ABC银行绩效积分管理系统
- **版本号**: v1.0
- **文档日期**: 2025年12月1日
- **技术栈**: Vue3 + UniApp + UniCloud
- **目标平台**: 微信小程序
- **预期用户数**: 50人（支行客户经理）

---

## 一、产品概述

### 1.1 产品定位
本系统是一款面向ABC银行支行客户经理的绩效积分管理工具，旨在将客户经理的任务完成情况线上化提报，自动转换为积分进行量化管理，实现业绩透明化、管理数字化。

### 1.2 核心价值
- **业务数字化**: 将线下纸质填报转为线上实时提报
- **激励可视化**: 积分实时计算，排行榜实时展示，增强工作动力
- **管理便捷化**: 管理员可随时查看、导出、调整业务数据
- **流程规范化**: 标准化的计分规则，公平公正的考核机制

---

## 二、产品背景与目标

### 2.1 业务背景
- 支行现有约50名客户经理，负责个贷和小微业务
- 当前绩效考核采用手工填报、季度汇总的方式，效率低下
- 缺乏实时反馈机制，客户经理无法及时了解自己的业绩排名
- 管理层难以实时掌握各支行、各客户经理的业务进展

### 2.2 产品目标
- **短期目标（1个月内）**: 
  - 完成系统上线，所有客户经理注册使用
  - 实现业务提报、积分统计、排行榜基础功能
  - 替代传统Excel填报方式

- **中期目标（3个月内）**:
  - 积累一个季度的完整业务数据
  - 优化计分规则和业务分类
  - 建立数据分析看板

- **长期目标（1年内）**:
  - 对接银行核心系统，实现数据自动同步
  - 扩展至全行范围使用
  - 增加业绩预测、智能分析等高级功能

## 四、功能需求详述

### 4.1 用户端功能

#### 4.1.1 登录认证
**页面**: Login.vue

**功能需求**:
1. **手机号密码登录**
   - 输入框：手机号（11位）、密码
   - 记住密码功能（可选）
   - 默认密码：123456（可由管理员预设）
   
2. **手机号验证码登录**
   - 输入框：手机号（11位）
   - 获取验证码按钮（60秒倒计时）
   - 验证码输入框（6位）
   
3. **微信小程序快速登录**
   - 点击"微信一键登录"
   - 获取微信授权信息（昵称、头像）
   - 与手机号进行绑定验证
   - 首次登录需绑定手机号
   
4. **忘记密码**
   - 通过验证码重置密码

**业务规则**:
- 手机号必须是管理员预先导入的白名单用户
- 密码错误超过5次，锁定账号10分钟
- 验证码有效期5分钟
- 微信登录需首次绑定手机号后才能使用

**技术实现**:
- UniCloud云函数：login
- 云数据库表：users
- 使用uni.login()获取微信code
- 使用uni.getUserProfile()获取用户信息

## 五、数据库设计（UniCloud云数据库）

### 5.1 表结构设计

#### users（用户表）
```javascript
{
  _id: string,              // 自动生成
  phone: string,            // 手机号（唯一索引）
  password: string,         // 密码（加密存储）
  name: string,             // 姓名
  branch: string,           // 所属支行
  role: string,             // 角色：manager/admin
  status: string,           // 状态：active/disabled
  wxOpenId: string,         // 微信OpenID（可选）
  wxUnionId: string,        // 微信UnionID（可选）
  wxNickname: string,       // 微信昵称（可选）
  wxAvatar: string,         // 微信头像（可选）
  createTime: timestamp,    // 创建时间
  lastLoginTime: timestamp, // 最后登录时间
  updateTime: timestamp     // 更新时间
}
```

**索引**:
- phone（唯一索引）
- wxOpenId（索引）
- branch（索引）

---

#### submissions（提报记录表）
```javascript
{
  _id: string,              // 自动生成
  userId: string,           // 用户ID（关联users._id）
  userName: string,         // 用户姓名（冗余，便于查询）
  userBranch: string,       // 用户支行（冗余）
  quarter: string,          // 所属季度（如：2025Q4）
  ruleId: string,           // 业务规则ID
  ruleName: string,         // 业务名称（冗余）
  ruleCategory: string,     // 业务条线：personal/micro
  type: string,             // 提报类型：new/stock
  count: number,            // 笔数
  amount: number,           // 授信金额（万元）
  score: number,            // 获得积分
  status: string,           // 状态：pending/approved/rejected
  reviewerId: string,       // 审核人ID（可选）
  reviewTime: timestamp,    // 审核时间（可选）
  reviewNote: string,       // 审核备注（可选）
  createTime: timestamp,    // 提报时间
  updateTime: timestamp,    // 更新时间
  updatedAt: timestamp      // 最后修改时间（用于追踪编辑历史）
}
```

**索引**:
- userId + quarter（联合索引）
- quarter（索引）
- createTime（索引）
- status（索引）

```javascript
{
  _id: string,              // 规则ID（如：p_agri_mortgage）
```

## 八、业务规则详述

### 8.1 积分计算规则

#### 个贷条线（Personal Loans）

| 业务类型 | 新增笔数积分 | 新增金额积分 | 存量笔数积分 | 存量金额积分 |
|---------|------------|------------|------------|------------|
| 农户抵押类 | 3分/笔 | 1分/100万 | 1.5分/笔 | 0分 |
| 经营抵押类 | 3分/笔 | 1分/100万 | 1.5分/笔 | 0分 |
| 商户e贷 | 1分/笔 | 1分/100万 | - | - |
| 消费贷 | 1分/笔 | 1分/100万 | - | - |
| 快农贷 | 1分/笔 | 1分/100万 | - | - |

#### 小微条线（Micro Business）

| 业务类型 | 新增笔数积分 | 新增金额积分 | 转贷笔数积分 | 转贷金额积分 |
|---------|------------|------------|------------|------------|
| 抵押类 | 3分/笔 | 1分/100万 | 1.5分/笔 | 0分 |
| 信用类-转贷 | - | - | 0.5分/笔 | 0分 |
| 微捷贷 | 1分/笔 | 1分/100万 | - | - |
| 闽e贷 | 1分/笔 | 1分/100万 | - | - |
| 线下-转贷 | - | - | 4.5分/笔 | 0分 |
| 科技贷 | 9分/笔 | 1分/100万 | - | - |
| 智动贷 | 9分/笔 | 1分/100万 | - | - |

**计算公式**:
```javascript
总积分 = 笔数积分 + 金额积分
笔数积分 = 笔数 × 对应规则笔数分值
金额积分 = (授信金额 / 100) × 对应规则金额分值
```

**示例**:
```
业务：农户抵押类-新增
笔数：2笔
金额：150万

计算：
笔数积分 = 2 × 3 = 6分
金额积分 = (150 / 100) × 1 = 1.5分
总积分 = 6 + 1.5 = 7.5分
```

### 8.2 排行榜规则
1. 按总积分降序排列
2. 积分相同时，按最后提报时间升序排列（早提报的排前面）
3. 仅统计客户经理，管理员不参与排名
4. 排名实时更新，任何人提报后立即刷新
5. 并列名次处理：
   - 如2人并列第2，则显示：第2名、第2名、第4名...

---

### 8.3 季度管理规则
1. **季度周期**：
   - Q1：1月1日 - 3月31日
   - Q2：4月1日 - 6月30日
   - Q3：7月1日 - 9月30日
   - Q4：10月1日 - 12月31日

2. **季度切换**：
   - 管理员手动切换
   - 切换时可选择是否归档旧数据
   - 归档后旧数据只读，不可修改

3. **季度清空**：
   - 清空所有客户经理的当前季度积分
   - 提报记录保留，但不计入新季度
   - 操作不可逆，需二次确认

---

### 8.4 数据审核规则
1. **默认模式**：提报后自动通过，积分立即生效
2. **审核模式**（可选）：
   - 管理员在系统设置中开启
   - 提报后状态为"待审核"
   - 管理员审核通过后积分才生效
   - 审核拒绝可填写拒绝原因
